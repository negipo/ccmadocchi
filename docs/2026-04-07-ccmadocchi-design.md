# ccmadocchi 設計仕様

Claude Codeのタスク完了時に、USB経由でぬいぐるみの腕をサーボモーターで動かす装置。

## 対象ぬいぐるみ

- アイドルマスター シャイニーカラーズ ミニぬいぐるみ 樋口円香
- 全高約16cm、腕の長さ約3cm
- 布製一体型（腕は胴体に縫い付けられている）

## ハードウェア構成

### 接続

- SG90の3本線（信号:オレンジ / 5V:赤 / GND:茶）をジャンパーワイヤー経由でMaker Nanoのピンに接続
  - 信号線 → デジタルピン9
  - 赤(VCC) → 5Vピン（USB給電から出力）
  - 茶(GND) → GNDピン
- ブレッドボード不要（ジャンパーワイヤーで直接接続）
- 外部電源・電池不要（SG90 1台の軽負荷はUSB給電で十分）

### 機構

- 市販スタンドでぬいぐるみを立たせる（スタンドは支え専用）
- SG90サーボをスタンドの横に固定（両面テープまたはホットボンド）
- サーボホーン（SG90に付属する十字型パーツ）にぬいぐるみの腕を結束バンドでくくりつける
- サーボの回転がそのまま腕の上下動になる

## ソフトウェア構成

### PC側（Python）

- CLIツール `ccmadocchi` として実装
- pyserialでMaker Nanoとシリアル通信
- サブコマンド: `ccmadocchi wave` で腕を動かす信号を送信
- インストール: `uv tool install .` で `~/.local/bin` に配置し、単独CLIとしてどこからでも実行可能
- 将来的にClaude Codeのhooksから呼び出す想定

### Maker Nano側（Arduino IDE + 標準Servoライブラリ）

- シリアル通信（9600baud）で信号を受信
- 信号を受けたらSG90を指定パターンで動かす
- 動きのパターン（角度、速度、往復回数）はパラメータとして定義し、後から調整可能にする
- ランダム性を取り入れたパターンに対応できる構造にする

### 通信プロトコル

- PC → Maker Nano: シリアルで1バイトのコマンド送信（例: `w` = wave）
- Maker Nano → PC: 動作完了後にACK返送（任意）

## 組み立て手順

1. Maker NanoにUSB Micro-Bケーブルを接続し、Arduino IDEでスケッチを書き込む
2. SG90の3本線をジャンパーワイヤー経由でMaker Nanoに接続（信号→D9、赤→5V、茶→GND）
3. ぬいぐるみをスタンドに立てる
4. SG90をスタンドの横に固定（両面テープ等）
5. サーボホーンにぬいぐるみの腕を結束バンドでくくりつける
6. `ccmadocchi wave` で動作確認
